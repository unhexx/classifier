# Архитектура

## Описание сервиса

Унифицированный сервис классификации с предварительной очисткой контекста от персональных данных (ПД) **локальной CPU-моделью**, интерфейсом контролёра для контроля очистки и механизмом **дообучения модели** на исправлениях контролёра.

## Общая схема

```
                    ┌─────────────────────┐
                    │  Интерфейс         │
                    │  контролёра (/ui)  │
                    └─────────┬───────────┘
                              │
┌──────────┐    ┌─────────────▼─────────────┐    ┌──────────────────┐
│ CLI/API  │───▶│      FastAPI (main)       │───▶│  SQLite (аудит)  │
└──────────┘    └─────────────┬─────────────┘    └──────────────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
     ┌────────────┐  ┌──────────────┐  ┌─────────────┐
     │ PD Cleaner │  │ Classifier   │  │  Training   │
     │ (CPU v1)   │  │ Engine       │  │  Service    │
     └─────┬──────┘  └──────┬───────┘  └──────┬──────┘
           │                │                  │
           ▼                ▼                  ▼
     ┌────────────┐  ┌──────────────┐  ┌─────────────┐
     │ Базовые    │  │ Catalog      │  │ Learned     │
     │ паттерны + │  │ Registry     │  │ Patterns    │
     │ дообученные│  │ (in-memory)  │  │ (из feedback)│
     └────────────┘  └──────────────┘  └─────────────┘
```

## Пайплайн обработки запроса

```
Входной контекст
      │
      ▼
┌─────────────────┐
│ 1. Очистка ПД   │  LocalPdCleaner (CPU, локально)
│    pd_cleaner   │  → entities[], cleaned_text
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. Классификация│  HybridScorer (keyword+fuzzy+trigram)
│    engine       │  → matches[] с confidence
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. Аудит        │  classification_logs, pd_cleaning_logs
└─────────────────┘
```

## Компоненты

### `app/core/pd_cleaner.py` — Локальная CPU-модель очистки ПД

- Детерминированные regex-паттерны для типовых российских ПД
- Загрузка **дообученных правил** из таблицы `learned_pd_patterns`
- Работает на CPU, без GPU и без сетевых вызовов
- Версия модели: `pd-cpu-v1` (расширяема)

### `app/core/classifier.py` — Классификатор неисправностей

Гибридный скорер на очищенном контексте:

| Сигнал | Вес | Назначение |
|--------|-----|------------|
| Keyword (Jaccard) | 0.35 | Совпадение токенов |
| Fuzzy (rapidfuzz) | 0.40 | Опечатки, вариации |
| Trigram overlap | 0.25 | Лексическое сходство |

### `app/services/training_service.py` — Дообучение

Цикл обратной связи контролёра:

```
Контролёр находит ошибку
        │
        ▼
POST /api/v1/feedback  →  controller_feedback (status: pending)
        │
        ▼
POST /feedback/{id}/apply  →  learned_pd_patterns (новое правило)
        │
        ▼
Следующая очистка использует новое правило
        │
        ▼
POST /feedback/export  →  data/training/pd_feedback.jsonl
```

### `app/static/ui.html` — Интерфейс контролёра

Одностраничное веб-приложение:

- Предпросмотр очистки ПД с визуализацией сущностей
- Классификация с отображением полного ответа API
- Форма обратной связи и очередь исправлений
- Экспорт данных для дообучения

### Хранилище (SQLite)

| Таблица | Назначение |
|---------|------------|
| `catalogs`, `faults` | Справочники неисправностей |
| `pd_cleaning_logs` | Журнал очистки ПД |
| `classification_logs` | Журнал классификаций (с ПД-метаданными) |
| `controller_feedback` | Исправления от контролёра |
| `learned_pd_patterns` | Дообученные правила очистки |

## Безопасность и локальность

- Все модели работают на CPU внутри процесса сервиса
- Исходный контекст с ПД **не покидает** систему (нет внешних API)
- `original_context` возвращается в ответе API только при обнаружении ПД — для контроля контролёром
- Журналы хранятся локально в SQLite

## Расширяемость

- Замена `LocalPdCleaner` на ML-модель (spaCy, transformers) — тот же интерфейс
- Новые типы ПД — через обратную связь контролёра или расширение `BASE_PATTERNS`
- Новые справочники — JSON в `data/catalogs/` + `make seed`
