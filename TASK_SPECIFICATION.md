# TASK_SPECIFICATION.md — Унифицированный сервис классификации типовых неисправностей

> **Источник истины.** Все задачи, приёмка и детали реализации берутся из этого файла.
> Версия: 1.0 (2026-06-24)

## 1. Цель проекта (из запроса пользователя)

Разработать **унифицированный сервис классификации**, который:

- Имеет **справочник** (каталог), содержащий **полный список возможных типовых неисправностей**.
- Содержит **все необходимые базы данных, данные, функции, модели, модули, компоненты, библиотеки, интерфейсы, датасеты** и т.д.
- Обеспечивает **быстрое развёртывание приложения** и **полное локальное функционирование "из коробки"**.
- Принимает через API:
  - данные контекста (описание проблемы, симптомы и т.п.)
  - наименование справочника
- Определяет **одну или несколько типовых неисправностей**.
- Возвращает **список** в ответе пользователю.

Сервис должен работать полностью локально, без внешних зависимостей в рантайме (после установки).

## 2. Ключевые требования

### Функциональные
- Поддержка **множества именованных справочников** (catalogs) по имени.
- Гибридная классификация контекста на основе:
  - ключевых слов и симптомов
  - нечёткого сопоставления (fuzzy)
  - (опционально) семантического сходства
- Возврат **ранжированного списка** (top-k) с оценкой уверенности (confidence 0..1).
- Минимум одна и максимум N неисправностей в ответе (настраивается top_k + min_confidence).
- Возможность расширения справочников (импорт/seed).

### Нефункциональные (критично для "из коробки")
- **Полностью локальная работа** после `pip install` или `docker run`.
- Время ответа классификации < 100 мс на типичном железе для каталога ~100 записей.
- Нет вызовов в интернет в процессе работы (кроме опциональной первой загрузки embedding-модели).
- Данные справочников встроены в образ/пакет (seed JSON + SQLite).
- Кросс-платформенность: Linux, Windows, macOS (через Docker — приоритет).

### Данные и справочники
- Минимум **2 полноценных справочника**:
  - `servers` — серверное и IT-оборудование (≥25 записей)
  - `network` — сетевое оборудование (≥20 записей)
- Каждая запись справочника содержит:
  - code, title, description
  - symptoms (массив)
  - keywords (массив)
  - category
  - recommended_actions (массив)
- Все тексты на русском языке (реалистичные формулировки из практики эксплуатации).

### Интерфейсы
- **REST API** (FastAPI + автоматическая OpenAPI/Swagger).
  - `POST /api/v1/classify`
  - `GET /api/v1/catalogs`
  - `GET /api/v1/catalogs/{name}/faults`
- **CLI**:
  - `unhexx-classifier classify --catalog <name> "<context>"`
  - `unhexx-classifier serve`
- **Python API** (импортируемая библиотека).

### Развёртывание
- `pip install -e ".[dev]"` → работающий CLI и возможность импорта.
- `docker build && docker run` → полностью рабочий сервис.
- `docker-compose up` — готовый демо-стенд с персистентной БД.
