# Справочники типовых неисправностей

JSON-файлы в этой директории — **источник истины** для справочников классификации обращений по типовым неисправностям.

## Принципы построения (на основе лучших практик индустрии)

Справочники построены на основе:
- ITIL (Incident Classification)
- FMEA (IEC 60812, SAE J1739) — failure modes
- CMMS (IBM Maximo, SAP PM) — failure code hierarchies (Problem / Cause / Remedy)
- Практики SRE и эксплуатации крупных дата-центров / производств

**Ключевые характеристики хорошей записи:**
- Чёткая категория (стандартизированная)
- Явный failure mode
- Богатые симптомы и keywords (реальные фразы из логов, индикаторы, разговорный язык инженеров)
- Конкретные, actionable recommended actions
- Реалистичные формулировки из практики

Подробный дизайн и таксономия — см. [docs/CATALOG_DESIGN.md](../docs/CATALOG_DESIGN.md).

## Формат записи

```json
{
  "code": "SRV-PSU-002",
  "title": "Краткое название неисправности",
  "description": "Что именно происходит",
  "failure_mode": "Полный отказ | Деградация | Интермиттент | ...",  // рекомендуется
  "symptoms": ["симптом 1", "ошибка 0x...", "красный индикатор"],
  "keywords": ["psu", "питание", "red light", "не включается"],
  "category": "Питание",
  "recommended_actions": [
    "Конкретное действие 1",
    "Проверка 2",
    "..."
  ],
  "meta": {
    "typical_cause": "опционально"
  }
}
```

## Стандартизированные категории (используются во всех каталогах)

- Питание
- Хранение данных
- Сетевое подключение / Связь
- Охлаждение и вентиляция
- Процессор, память, контроллеры
- Материнская плата / Аппаратная платформа
- Конфигурация, прошивка и ПО
- Маршрутизация и L3
- L2 / Switching
- Мониторинг, управление и сервисы
- Производительность и деградация
- Безопасность и доступ
- Виртуализация
- Резервное копирование и восстановление

## Доступные справочники

| Файл | Имя | Записей (цель) | Описание |
|------|-----|----------------|----------|
| `servers.json` | servers | 80+ | Серверное и IT-оборудование (стойки, блейды, вычислительные хосты) |
| `network.json` | network | 50+ | Сетевое оборудование (коммутаторы, маршрутизаторы, трансиверы) |
| `automotive.json` | automotive | 80+ | Автомобильные неисправности |
| `industrial.json` | industrial | 50+ | Промышленное оборудование (ИБП, HVAC, ДГУ, насосы, компрессоры) |
| `facilities.json` | facilities | 55+ | Инфраструктура зданий и дата-центров (питание, охлаждение, пожаротушение, экология) |

## Загрузка и применение

```bash
make seed
# или
python scripts/seed.py
```

Сиды загружаются автоматически при старте (идемпотентный upsert по `code`).

## Добавление / изменение справочника

1. Следуй принципам из `CATALOG_DESIGN.md`
2. Создай/отредактируй `<name>.json`
3. Запусти `make seed`
4. Проверь качество классификации через `/api/v1/classify` и `/ui`

## Валидация

Все справочники должны проходить проверку:
- Обязательные поля
- Уникальность `code` внутри каталога
- Наличие содержательных `symptoms` и `recommended_actions`

См. тесты и будущие скрипты валидации.
